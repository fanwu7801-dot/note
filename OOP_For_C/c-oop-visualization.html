📦 结构体封装 - C语言的"类"

🎯 核心概念

在C语言中，结构体就是我们的"类"。通过结构体，我们可以将数据成员和函数指针封装在一起，实现面向对象的封装特性。

// LED驱动"类"的定义
typedef struct {
    // 属性成员（相当于类的成员变量）
    led_drv_attr_t    attr;           // LED属性
    
    // 方法成员（通过函数指针实现）
    led_drv_ops_t     *p_ops;         // 硬件操作接口
    led_drv_time_t    *p_time_base;   // 时间服务接口
    
    // 成员方法
    led_drv_status_t  (*pf_led_controler)(
        struct bsp_led_drv_t  *const self,
        led_drv_attr_t        *const attr
    );
} bsp_led_drv_t;

🔍 结构体成员解析（点击查看）

led_drv_attr_t attr - LED属性配置
包含：闪烁次数、周期时间、占空比等参数
led_drv_ops_t *p_ops - 硬件操作接口
指向LED开关操作的函数指针表
(*pf_led_controler) - 控制方法
LED控制的核心方法，注意第一个参数是self指针！

🔗 函数指针 - 实现多态的魔法

🎪 函数指针的魔力

函数指针是C语言实现面向对象多态性的关键。通过函数指针，我们可以在运行时动态绑定不同的函数实现。

// 操作接口定义（类似于接口或抽象类）
typedef struct {
    void (*pf_led_on)(void);   // LED开启方法
    void (*pf_led_off)(void);  // LED关闭方法
} led_drv_ops_t;

// 时间服务接口
typedef struct {
    void (*pf_delay_ms)(uint32_t ms);  // 延时方法
} led_drv_time_t;

👤 Self指针 - C语言的"this"

🎯 Self指针的作用

在面向对象语言中，我们有this指针指向当前对象。在C语言中，我们通过显式传递self指针来实现相同的功能。

// 注意每个"成员函数"的第一个参数都是self指针
static led_drv_status_t led_controler(
    bsp_led_drv_t * const self,  // 相当于this指针
    led_drv_attr_t * const attr
)
{
    // 通过self访问对象的成员
    self->attr.blink_times = attr->blink_times;
    self->attr.cycle_time_ms = attr->cycle_time_ms;
    
    // 调用其他成员方法
    ret = led_blink(self);
    
    return ret;
}
🔑

关键要点

  • 📌 Self指针必须显式传递
  • 🔐 通常声明为const，防止意外修改指针本身
  • 🎯 通过self访问所有成员变量和方法
  • 🔄 支持方法链式调用

🏭 对象实例化 - "构造函数"

🎨 初始化过程

C语言没有构造函数，但我们可以通过初始化函数来模拟构造过程，完成对象的创建和初始化。

// "构造函数" - 实例化LED驱动对象
led_drv_status_t bsp_led_drv_inst(
    bsp_led_drv_t    * const self,      // 要初始化的对象
    led_drv_ops_t    * const ops,       // 注入的硬件操作接口
    led_drv_time_t   * const time_base  // 注入的时间服务
)
{
    // 1. 参数检查（防御性编程）
    assert(NULL != self || NULL != ops || NULL != time_base);
    
    // 2. 检查资源状态
    if(true == self->attr.is_inited) {
        return LED_DRV_ERR_RESOURCE;
    }
    
    // 3. 依赖注入 - 绑定接口
    self->p_ops = ops;
    self->p_time_base = time_base;
    
    // 4. 绑定成员方法
    self->pf_led_controler = led_controler;
    
    // 5. 初始化属性
    self->attr.blink_times = 0;
    self->attr.cycle_time_ms = 0;
    self->attr.is_inited = true;
    
    return LED_DRV_OK;
}

📋 初始化步骤

步骤1：参数验证

确保所有必需的参数都有效

步骤2：状态检查

防止重复初始化

步骤3：依赖注入

注入外部依赖，实现解耦

步骤4：方法绑定

绑定对象的方法实现

步骤5：属性初始化

设置默认值，标记初始化完成

// 使用示例代码
// 1. 创建LED对象
bsp_led_drv_t led1, led2, led3;

// 2. 准备硬件接口和时间服务
led_drv_ops_t led_ops = {
    .pf_led_on = GPIO_LED_ON,
    .pf_led_off = GPIO_LED_OFF
};

led_drv_time_t time_service = {
    .pf_delay_ms = HAL_Delay
};

// 3. 初始化LED对象（调用"构造函数"）
bsp_led_drv_inst(&led1, &led_ops, &time_service);
bsp_led_drv_inst(&led2, &led_ops, &time_service);
bsp_led_drv_inst(&led3, &led_ops, &time_service);

// 4. 配置LED参数
led_drv_attr_t config = {
    .blink_times = 5,        // 闪烁5次
    .cycle_time_ms = 1000,   // 周期1秒
    .proportion_on_off = 50  // 50%占空比
};

// 5. 控制LED
led1.pf_led_controler(&led1, &config);  // 注意传递self指针！
