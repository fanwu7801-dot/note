##  为什么嵌入式开发需要使用oop思想？
### 理解什么是面相对象和面相过程
- 面向对象:内聚数据和行为，在和构建数据之间的联系过程顺便解决问题
- 面向过程:关注的是完成任务的步骤和过程，数据和行为是分离的
> 面向对象更符合人类的思维方式，更容易理解和维护复杂系统,例如汽车的例子
> 生产汽车时，面向过程关注的是制造汽车的步骤，而面向对象则关注汽车作为一个整体，包括它的各个部件和功能，使得设计和维护更加直观和高效。
> 面向对象最好的应用场景是复杂系统的设计和维护，例如大型软件系统、游戏开发和嵌入式系统等。

#### 面向对象的四大特性
1. 封装:将数据和行为封装在对象内部，隐藏实现细节，只暴露必要的接口
2. 继承:子类可以继承父类的属性和方法，促进代码重用
3. 多态:同一接口可以有不同的实现方式，提高代码的灵活性
4. 抽象:通过抽象类和接口定义对象的共同行为，简化复杂系统的设计
> 例如，汽车类可以有不同的子类，如电动汽车和燃油汽车，它们继承了汽车类的属性和方法，但实现方式不同，这就是多态的体现。

### 结构体封装数据和行为

##### 函数指针
![alt text](embedded-development-note-main/OOP_For_C/image-1.png)
遥控器就像函数指针，不同的遥控器可以控制不同的设备，但它们都遵循相同的接口规范。
**函数在内存中也是有地址的**，函数指针存储了函数的地址，通过函数指针可以调用不同的函数实现，从而实现多态。
__函数指针的本质__：存贮函数的地址的变量
**语法：**
![alt text](embedded-development-note-main/OOP_For_C/image-2.png)
- int (*func_ptr)(int, int); // 定义一个函数指针，指向返回类型为int，参数为两个int的函数
- int *---->*返回类型
- (*func_ptr) *---->*函数指针变量名
- (int, int) *---->*函数参数列表
**给函数指针赋值：**

```c
int add(int a, int b) {
    return a + b;
}
int (*func_ptr)(int, int) = add; // 将函数指针指向add函数
```
>函数名本身就是函数的地址，可以直接赋值给函数指针变量。
**通过函数指针调用函数：**

方法一：**解引用**
```c
int result = (*func_ptr)(3, 4); // 调用add函数，结果为7
```
在C语言当中,函数调用运算符(),对"函数指针"会自动进行一次解引用,所以这两种做法是等价的.本质上来说,这是C的特性(隐式解引用);
方法二：**直接调用**
```c
int result = func_ptr(3, 4); // 调用add函数，结果为7
```
> 现代C更倾向于直接调用函数指针，更简洁。

``` cpp
#include <stdio.h>

int add (int a, int b) {
    return a + b;
}
int sum (int a, int b) {
    return a - b;
}

/*main函数当中声明函数指针*/
int main()
{
    int (*p)(int , int );  /*声明一个函数指针p，指向函数，函数的参数为两个int类型，返回值为int类型*/
    p = add;               /*让函数指针p指向add函数*/
    int result = p(3,4);  /*通过函数指针调用add函数*/
    printf("result = %d\n",result);
    /*实现动态绑定*/
    p = sum;              /*让函数指针p指向sum函数*/
    result = p(3,4);      /*通过函数指针调用sum函数*/
    printf("result = %d\n",result);

    return 0;
}
```
常见错误：
1. 忘记加括号：`int *func_ptr(int, int);` 这是一个函数声明，返回一个int指针，而不是函数指针。
2. 参数类型不匹配：确保函数指针的参数类型和返回类型与实际函数匹配。
3. 未初始化函数指针：在使用函数指针之前，必须先将其指向一个有效的函数，否则会导致未定义行为。

### 函数指针和结构体结合实现面向对象
> 当然结构体不能直接放函数，但是可以放函数指针，通过函数指针实现结构体的方法，从而实现面向对象的特性，这里就是封装了
```c
struct Dog
{
    char* name;
    int age;
    void (*bark)(); // 函数指针作为方法
    void (*eat)(char * food); // 设置年龄方法
}
```
> **注意** ：函数指针作为结构体成员时，必须确保函数指针的签名（参数和返回类型）与实际函数匹配。
``` c
void bark_function() {
    printf("Woof! Woof!\n");
}
void eat_function(char* food) {
    printf("Eating %s\n", food);
}

int main() {
    struct Dog myDog;
    myDog.name = "Buddy";
    myDog.age = 3;
    myDog.bark = bark_function; // 赋值函数指针
    myDog.eat = eat_function;   // 赋值函数指针
    printf("Dog Name: %s, Age: %d\n", myDog.name, myDog.age);
    myDog.bark();               // 调用bark方法
    myDog.eat("bone");         // 调用eat方法
    return 0;
}
```

#### 高内聚低耦合的概念

“高内聚低耦合”是软件工程中的一个核心设计原则，强调模块（或类/函数）的良好组织方式。它是衡量代码质量的重要标准，尤其在嵌入式开发中，能提高代码的可维护性、可重用性和可扩展性。

##### 1. 内聚（Cohesion）：模块内部的紧密程度
- **定义**：内聚衡量一个模块（函数、类或结构体）内部各部分之间的相关性和依赖程度。
- **高内聚**：模块内的所有元素（变量、函数）都紧密相关，专注于单一职责。例如，一个LED控制模块只负责LED的开关和状态管理，不掺杂其他功能。
- **低内聚（坏例子）**：模块内部元素松散无关，导致功能混乱。例如，一个函数既处理LED控制，又处理串口通信和数据存储。

**好处**：高内聚让模块易理解、易测试、易修改。

##### 2. 耦合（Coupling）：模块之间的依赖程度
- **定义**：耦合衡量不同模块之间的相互依赖程度。
- **低耦合**：模块之间依赖少，通过接口（函数调用或消息）交互，而不是直接访问内部数据。例如，LED模块通过函数接口提供服务，其他模块只需调用接口，无需知道内部实现。
- **高耦合（坏例子）**：模块之间紧密依赖，一个模块的改变会影响多个其他模块。例如，全局变量被多个模块直接修改，导致“牵一发而动全身”。

**好处**：低耦合让模块独立，易于替换、测试和重用。

##### 3. 为什么在嵌入式开发中重要？
- **可维护性**：高内聚让修改局部化，低耦合减少修改的连锁反应。
- **可重用性**：低耦合的模块可以轻松移植到其他项目。
- **可扩展性**：新功能可以作为独立模块添加，而不影响现有代码。
- **调试便利**：问题局限在特定模块内。

##### 4. 在C语言嵌入式开发中的应用
结合面向对象思想，可以通过结构体和函数指针模拟高内聚低耦合：

```c
// 高内聚：LED模块只负责LED相关功能
typedef struct {
    uint8_t state;  // 内部数据，封装
    void (*set_state)(struct led_device* dev, uint8_t state);  // 接口
} led_device_t;

// 低耦合：其他模块通过接口调用，不直接访问内部
void some_other_module() {
    led_device_t led;
    led.set_state(&led, 1);  // 只依赖接口，不关心实现
}
```

- **避免全局变量**：全局变量增加耦合，应尽量用局部变量或传递参数。
- **接口抽象**：用函数指针定义接口，让实现可替换（多态）。
- **模块化**：将代码分成独立文件，每个文件负责一个功能。

##### 5. 如何实现高内聚低耦合？
- **设计时**：每个模块问自己“这个功能属于这里吗？”（内聚）和“这个模块依赖其他模块吗？”（耦合）。
- **重构**：定期检查代码，如果一个函数做太多事，拆分它；如果模块间依赖太多，引入接口层。
- **工具**：在嵌入式中，可以用静态分析工具检查依赖关系。
### this指针模拟类的实例
Self指针 - C语言的"this"指针

在C语言中模拟面向对象时，没有内置的 `this` 指针。我们通过在函数参数中显式传递一个指向当前对象的指针，通常命名为 `self`，来模拟 `this`。

#### self 指针的作用
- **定义**：`self` 是一个指向结构体实例的指针，作为函数的第一个参数传入。
- **作用**：
  - 允许“方法”访问对象的成员变量和方法。
  - 实现封装和多态（通过函数指针）。
  - 相当于面向对象语言中的 `this`，但需要手动传递。

#### 示例：UART设备模拟
```c
typedef struct UART_Device {
    uint32_t baudrate;  // 成员变量
    void (*set_baudrate)(struct UART_Device* self, uint32_t baudrate);  // 方法
} UART_Device;

// 方法实现（self 作为第一个参数）
void uart_set_baudrate(UART_Device* self, uint32_t baudrate) {
    self->baudrate = baudrate;  // 通过 self 访问成员
}

// 创建实例
UART_Device dev;
dev.set_baudrate = uart_set_baudrate;  // 绑定方法
dev.set_baudrate(&dev, 9600);  // 调用时传递 &dev 作为 self
```

- **对比C++**：
  - C++：`this` 隐式存在，`this->baudrate = baudrate;`
  - C：`self` 显式传递，`self->baudrate = baudrate;`

#### 注意事项
- 总是检查 `self` 不为 NULL（用断言：`assert(self != NULL);`）。
- 在多线程环境中，确保 `self` 的线程安全。
- `self` 允许方法“认识”自己在操作哪个对象，实现对象的行为。

``` c
static led_drv_status_t led_control(
    bsp_led_device_t* const self, // this指针
    const bsp_led_cmd_t cmd,
)
{
    // 通过self 访问成员的对象
    self->cmd_state = cmd;
    //调用成员函数
    self->set_state(self, cmd);
}
``` 
self 指针其实是指向当前的LED对象，并且self指针作为第一个参数传递给成员函数，这样成员函数就可以通过self指针访问和修改对象的属性，实现类似于面向对象中的"this"指针的功能。(注意：C语言没有内置的this指针，这只是通过约定俗成的方式来模拟面向对象的行为。)
> 这里的self必须是const类型，表示指针本身不可修改，但指针指向的内容是可以修改的。

#### 对象实例化
在C语言中，可以通过动态内存分配（如malloc）来创建对象实例
```c

{

    if(true == self->is_init)
    {
        return LED_DRV_STATUS_ERR_ALREADY_INIT;
    }
    // 3. 依赖注入 - 绑定接口和配置参数到对象实例
    self->ops = ops;
    self->cfg = cfg;
    // 4.绑定成员方法
    self->pf_led_control = led_control;
    // 5.初始化属性
    self->timer_id = BSP_INVALID_TIMER_ID;
    self->is_init = true;

}
```

#### *第一步：定义UART设备结构体*
在写代码之前，先要考虑UART设备有哪一些熟悉和方法，然后将它们封装在结构体中
> 属性成员:波特率、数据位、停止位、校验位,发送缓冲区，接收缓冲区等
> 方法成员:初始化、发送数据、接收数据等
```c
// 定义UART设备结构体
ifndef UART_DEVICE_H
#define UART_DEVICE_H
#include "stm32f4xx.h" // 假设使用STM32 HAL库
#include <stdint.h>
/*前向声明*/
typedef struct uart_device_t uart_device_t;
// 类的具体实现
struct uart_device_t {
    // 属性成员
    USART_TypeDef* hw; // UART硬件资源指针
    uint32_t baud_rate; // 波特率
    uint8_t data_bits;  // 数据位
    uint8_t stop_bits;  // 停止位
    uint8_t parity;     // 校验位
    uint8_t* tx_buffer; // 发送缓冲区
    uint8_t* rx_buffer; // 接收缓冲区
    // 方法成员（函数指针）
    void (*init)(uart_device_t* self); // 初始化方法
    void (*send)(uart_device_t* self, const uint8_t* data, uint16_t length); // 发送数据方法
    void (*receive)(uart_device_t* self, uint8_t* buffer, uint16_t length); // 接收数据方法
    void (*deinit)(uart_device_t* self); // 反初始化方法
};
#endif // UART_DEVICE_H
```
#### *第二步：实现UART设备的方法*
接下来实现结构体中的方法，这些方法将操作UART设备的属性和行为
```c
#include "uart_device.h"
#include <string.h> // 用于memcpy

static void uart_init_impl(uart_device_t* self) {
    // 1. 使能UART外设时钟
    if(self->hw == USART1) 
    {
        RCC->APB2ENR |= RCC_APB2ENR_USART1EN;
    }
    // 2. 配置UART参数（波特率、数据位、停止位、校验位）
    uint32_t usartdiv = SystemCoreClock / self->baud_rate*16;
    slef->hw->BRR = usartdiv;
    slef->hw->CR1 = USART_CR1_TE | USART_CR1_RE | USART_CR1_UE; // 使能发送、接收和UART
    //3. 清空缓冲区
    memset(self->tx_buffer, 0, sizeof(self->tx_buffer));
    memset(self->rx_buffer, 0, sizeof(self->rx_buffer));
}

static void uart_send_impl(uart_device_t* self, const uint8_t* data, uint16_t length) {
    for(uint16_t i = 0; i < length; i++) {
        while(!(self->hw->SR & USART_SR_TXE)); // 等待发送缓冲区空
        self->hw->DR = data[i]; // 发送数据
        while(!(self->hw->SR & USART_SR_TC)); // 等待传输完成
    }
}

uart_device_t* uart_create(
    USART_TypeDef* hw,
    uint32_t baud_rate
) 
{
    //1. 分配内存
    UATR_Device_t* uart = (uart_device_t*)malloc(sizeof(uart_device_t));
    if(uart == NULL) return NULL; // 内存分配失败
    //2. 初始化属性
    uart->hw = hw;
    uart->baud_rate = baud_rate;
    memset(uart->tx_buffer, 0, 256);
    memset(uart->rx_buffer, 0, 256);
    //3. 绑定方法
    uart->init = uart_init_impl;
    uart->send = uart_send_impl;
    uart->receive = uart_receive_impl;
    uart->deinit = uart_deinit_impl;
    //4. 返回实例
    return uart;
}
```

### 不透明指针和接口封装
1. 不透明指针（Opaque Pointer）是一种编程技术，用于隐藏数据结构的实现细节，只暴露接口给用户。这种方法有助于实现封装和信息隐藏，提高代码的可维护性和安全性。
2. 在C语言中，不透明指针通常通过在头文件中声明一个不完整的结构体类型来实现，而在源文件中定义该结构体的完整内容。
#### 不透明指针 a poke pointer
```c
typedef struct uart_device_t uart_device_t; // 前向声明，不完整类型
uart_device_t* uart_create(...); // 创建函数
void uart_destroy(uart_device_t* dev); // 销毁函数
```
> 这里最重要的就是 使用` typedef strcut uart_device_t uart_device_t;` 这一步进行前向声明,然后相当于对结构体 uart_device_t 进行了一次封装 ,导致了后续调用的时候能够直接使用提供的接口API进行更改数据,而不能使用 uatr_device_t->bind 去更改结构体成员. 
#### 接口设计
_1) `counter.h`（对外接口：只有不透明指针 + 方法）_
```  cpp
#ifndef COUNTER_H
#define COUNTER_H

#include <stddef.h>

#ifdef __cplusplus
extern "C" {
#endif

// 不透明类型：外部只知道它是一个类型，但不知道内部成员
typedef struct Counter Counter;

// “方法表”：用函数指针模拟对象方法（类似 C++ vtable）
typedef struct CounterOps {
    void (*destroy)(Counter* self);

    void (*add)(Counter* self, int delta);
    int  (*get)(const Counter* self);
    void (*reset)(Counter* self);

    // 把对象状态打印到 out（更通用）
    void (*print)(const Counter* self, char* out, size_t out_cap);
} CounterOps;

// “对象基类”对外可见部分：只暴露 ops 指针
struct Counter {
    const CounterOps* ops;
};

// 工厂函数（构造）
Counter* counter_create(const char* name, int initial);

#ifdef __cplusplus
}
#endif

#endif // COUNTER_H

```
_2)`counter.c`（内部实现：真正的 struct + 方法实现 + ops 表）_

``` cpp
#include "counter.h"

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

// 真实对象的“私有”数据：仅在 .c 内可见
typedef struct CounterImpl {
    Counter base;          // 必须放一个“基类头”，让外部能拿到 ops
    char* name;
    int value;
} CounterImpl;

// ---------- 方法实现（注意签名与 CounterOps 匹配） ----------

static void counter_destroy(Counter* self) {
    if (!self) return;
    CounterImpl* impl = (CounterImpl*)self;
    free(impl->name);
    free(impl);
}

static void counter_add(Counter* self, int delta) {
    if (!self) return;
    CounterImpl* impl = (CounterImpl*)self;
    impl->value += delta;
}

static int counter_get(const Counter* self) {
    if (!self) return 0;
    const CounterImpl* impl = (const CounterImpl*)self;
    return impl->value;
}

static void counter_reset(Counter* self) {
    if (!self) return;
    CounterImpl* impl = (CounterImpl*)self;
    impl->value = 0;
}

static void counter_print(const Counter* self, char* out, size_t out_cap) {
    if (!out || out_cap == 0) return;

    if (!self) {
        snprintf(out, out_cap, "Counter(NULL)");
        return;
    }

    const CounterImpl* impl = (const CounterImpl*)self;
    snprintf(out, out_cap, "Counter{name=\"%s\", value=%d}", impl->name ? impl->name : "(null)", impl->value);
}

// ---------- 方法表（vtable） ----------
static const CounterOps COUNTER_OPS = {
    .destroy = counter_destroy,
    .add     = counter_add,
    .get     = counter_get,
    .reset   = counter_reset,
    .print   = counter_print
};

// ---------- 构造函数 ----------
Counter* counter_create(const char* name, int initial) {
    CounterImpl* impl = (CounterImpl*)calloc(1, sizeof(CounterImpl));
    if (!impl) return NULL;

    impl->base.ops = &COUNTER_OPS;
    impl->value = initial;

    if (name) {
        impl->name = (char*)malloc(strlen(name) + 1);
        if (!impl->name) {
            free(impl);
            return NULL;
        }
        strcpy(impl->name, name);
    }

    return (Counter*)impl;
}
```

``` cpp
    c->ops->reset(c);
    c->ops->print(c, buf, sizeof(buf));

    // 析构
    c->ops->destroy(c);
```

```c
void uart_send(uart_device_t* self, const uint8_t* data, uint16_t length) {
    // 发送数据的具体实现
    if(self == NULL) return; // 检查指针有效性
    for(uint16_t i = 0; i < length; i++) {
        while(!(self->hw->SR & USART_SR_TXE)); // 等待发送缓冲区空
        self->hw->DR = data[i]; // 发送数据
        while(!(self->hw->SR & USART_SR_TC)); // 等待传输完成
    }
}
``` 

### 构造函数和析构函数
🚀在C语言中，虽然没有内置的构造函数和析构函数的概念，但我们可以通过自定义函数来模拟这些功能，以实现对象的初始化和清理
#### 构造函数
构造函数用于创建和初始化对象实例，通常会分配内存并设置初始状态
```c
uart_device_t* uart_create(USART_TypeDef* hw, uint32_t baud_rate) {
    uart_device_t* uart = (uart_device_t*)malloc(sizeof(uart_device_t));
    if(uart == NULL) return NULL; // 内存分配失败
    // 初始化属性
    uart->hw = hw;
    uart->baud_rate = baud_rate;
    // 绑定方法
    uart->init = uart_init_impl;
    uart->send = uart_send_impl;
    uart->receive = uart_receive_impl;
    uart->deinit = uart_deinit_impl;
    return uart;
}
```
#### 析构函数
析构函数用于清理对象实例，释放资源和内存
```c
void uart_destroy(uart_device_t* self) {
    if(self != NULL) {
        // 释放其他资源（如果有）
        free(self); // 释放内存
    }
}
```
