由一个HOST和一个或者多个Controller组成

SM:安全管理层，负责处理安全管理协议，也就是蓝牙设备的加密，认证等操作,也就是连接的时候生产，需要通过秘钥来实现。
GATT:通用化属性协议层文件，负责处理GATT协议，也就是蓝牙设备的属性访问操作

#### GATT的角色

1. GATT的作用
>
> - 定义了数据如何的组织(服务，特征，描述符)
> - 规定了数据如何的交互(读，写，通知，指示)
> - 确保了厂商设备之间的互操作性
>
2. GATT的角色

|角色|说明|
|---|---|
|Client|发起数据交互的设备，如手机|
|Server|提供数据交互服务的设备，如智能手表|

> 一个设备可以同时是Client和Server

##### 服务

服务的定义
>
> - 服务是一组相关功能的集合例如
>
- 心率服务，包含心率测量，传感器位置等数据
- 电池服务，包含电池电量，充电状态等
- 每一个服务都有一个唯一的uuid标识
- 标准uuuid：例如0x188d心率服务
- 自定义uuid：开发者可以自定义128bit的uuid，确保与其他厂商服务不冲突

``` css
GATT Server
│
├── Service 1 (UUID: 0x180D)  // 心率服务
│   ├── Characteristic 1 (心率测量)
│   └── Characteristic 2 (传感器位置)
│
└── Service 2 (UUID: 0x180F)  // 电池服务
    ├── Characteristic 1 (电池电量)
    └── Descriptor (通知使能)
```

##### 特征值

特征值的定义
> 特征值是服务当中的具体数据项
>
>- 心率服务当中的当前心率
>- 电池服务当中剩余电量
>- 每一个特征值都包含以下属性：
    - UUID（唯一标识符）：每个特征值都有一个唯一的UUID，用于识别和访问该特征值。
    - Properties（属性）：定义了特征值能做什么（读、写、通知等）。
    - Value（值）：数据本身，比如一个温度读数（25.5）或一个开关状态（0x01）。
    - Descriptor（描述符）：最常用的是CCCD，用于启用/禁用“通知”（Notify）或“指示”（Indicate）功能。

___特征值的属性___

|属性|说明|
|---|---|
|Read（读取）|允许Client读取特征值的当前值。例如电池电量等|
|Write（写入）|允许Client写入新值到特征值。例如改变LED的状态等|
|Notify（通知）|当特征值的值发生变化时，Server可以通过通知Client来告知Client。|
|Indicate（指示）|与通知类似，但Server必须确认Client收到了通知。类似于通知，但要求Client确认。|

___描述符___
作用> 描述符是特征值的补充，用于提供额外的信息或配置。
例如：

- CCCD（Client Characteristic Configuration Descriptor）：用于配置Client是否接收特征值的通知或指示。
- 权限描述符（Permission Descriptor）：定义了对特征值的访问权限，如只读、读写等。
常见的描述符

|UUID|名称|功能|
|---|---|---|
|0x2902|Client Characteristic Configuration Descriptor|配置Client是否接收特征值的通知或指示|
|0x2903|Permission Descriptor|定义了对特征值的访问权限|
